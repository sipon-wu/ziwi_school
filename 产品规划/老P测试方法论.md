# 老P 测试方法论 & 质量标准

> **目标**：QA 团队必须理解并内化这套方法，而非机械执行用例。
> **核心理念**：你不是在"验证功能"，你是在模拟真实教师使用这个产品。

---

## 一、三层测试视角

### L1：角色场景化流程视角

> 把自己当成张老师，从打开电脑到下班关机的完整一天。

| 时间段 | 场景 | 路径链 |
|--------|------|--------|
| 08:00 到校 | 看工作台了解今天待办 | 登录 → /dashboard |
| 08:10 备课 | 打开昨天保存的教案继续编辑 | /dashboard → /lesson-plans → 某个教案编辑 |
| 08:30 出题 | 基于教案知识点出一套课堂练习 | 教案页 → "基于此教案出题" → 选用途 → 生成 |
| 09:00 布置 | 把题目发布到班级 | 出题页 → 保存 → 发布 → 选班级 |
| 14:00 批阅 | 改作业、调分数、写评语 | /dashboard → /grading → 某份作业 |
| 17:00 看数据 | 查看今天全班学情 | /dashboard → /analytics |

**测试要点**：不是孤立测每个页面，而是**沿着这个链条走**。重点检查：
- 前一步的选择（知识点、年级、教材）是否**无损传递**到下一步？
- 后一步能否**回溯**到前一步修改？
- 频繁切换页面时，状态是否**保持**还是**丢失**？

### L2：UE 易用性视角

> 不是找 Bug，是找"让人不爽"的地方。

| 检查维度 | 具体标准 |
|----------|----------|
| **空间效率** | 一次性决策完成后，UI 是否及时回收空间？（命题用途、模板选择、引导步骤） |
| **信息可见性** | 全屏/弹窗模式下，原页面的操作条（已选、搜索、筛选）是否还在？还是"藏起来找不到了"？ |
| **操作连续性** | 连续操作（连续点击多个知识点、连续拖拽面板）时，之前的操作结果是否被意外重置？ |
| **反馈一致性** | 同样的操作（生成、保存、删除）在不同页面的反馈是否一致？（加载动画、成功提示、错误提示） |
| **死胡同检测** | 任何弹窗/面板/浮层，是否有明确的关闭途径？（X 按钮、点遮罩、ESC） |

### L3：产品逻辑视角

> 不是 UI 层面的问题，是"这个功能到底解决了什么问题"。

| 检查维度 | 具体标准 |
|----------|----------|
| **步骤不可逆性** | 某个操作一旦完成是否应该允许回退？（命题用途：选了就不能退 → 但允许按[修改]重新选。定稿教案：定稿后还能编辑吗？） |
| **默认值合理性** | AI 推荐/默认值是否合理？（首次进入新学科，推荐什么？题库为空时推荐什么？） |
| **边界完整性** | 极端场景下功能是否仍然可用？（0 个知识点时还能出题吗？1 个学生时学情分析有意义吗？） |

## 二、测试优先级矩阵

> QA 资源有限时，按这个优先级分配测试时间。

| 优先级 | 定义 | 典型场景 |
|--------|------|----------|
| **P0 阻断** | 用户核心任务无法完成 | 教案生成失败、出题按钮不响应、作业无法发布 |
| **P1 严重** | 核心任务能完成但体验极差 | 图谱点击后缩放归零、用途卡片占据半屏不消失、已选知识点在弹窗中消失 |
| **P2 一般** | 非核心路径问题 | 辅助面板显示不全、颜色不统一、间距不一致 |
| **P3 建议** | 影响美观但不影响使用 | 字体大小微调、动画不流畅 |

**P1 是 QA 最应该投入的**——这类问题不会被"按用例测"发现，必须通过"角色扮演"发现。

## 三、老P 发现问题的思维模型

```
普通 QA 思维：
  打开页面 → 检查元素是否存在 → 点击按钮 → 检查结果 → 通过 ✓

老P 思维：
  打开页面 → 这是什么场景？ → 上一个操作的结果有没有带过来？ → 
  我下一个操作是什么？ → 这个区域我不用了，为什么还占着位置？ → 
  我缩放了一下画布，点个节点会不会给我重置？ → 果然重置了 → Bug!
```

**核心差异**：老P 不是在"执行测试用例"，而是在**模拟真实使用**时，凭借对产品定位和 UE 规范的理解，识别出"不应该这样"的瞬间。

## 四、每次测试必做的 6 件事

| # | 事项 | 耗时 |
|---|------|------|
| 1 | 通读 `QA测试必检清单.md`，逐项打勾 | 5min |
| 2 | 走完一条完整的教师日场景链（工作台→备课→出题→批阅→学情） | 15min |
| 3 | 在宽屏（1920）和笔记本（1366）各走一遍 | 10min |
| 4 | 每种弹窗/面板都检查：有无关闭途径？内容是否完整？ | 5min |
| 5 | 频繁进行状态切换（学科/年级/布局/维度），检查是否引起副作用 | 5min |
| 6 | 记录任何"感觉不对"的瞬间（哪怕说不出原因），写进报告 | 随时 |
