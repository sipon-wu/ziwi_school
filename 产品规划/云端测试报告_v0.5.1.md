## 六、发现的问题

### 🔴 P0 阻断：AI 生成功能不可用

**现象**：教案生成/出题 API 返回 `{success:true, content:"[API错误] "}`

**根因**：百炼 DashScope API 返回 404：
```
docker exec zhiwei-demo-ai → dashscope.Generation.call(model='qwen-plus')
→ status_code: 404, Unknown error
```

**排查结果**：
- API Key 已配置：`DASHSCOPE_API_KEY=sk-ws-H.RPLRHHI...`
- 工作空间端点：`DASHSCOPE_BASE_URL=https://ws-brgw6hkhvu204ezg.cn-beijing.maas.aliyuncs.com`
- Python AI 服务正常启动（uvicorn running），qwen_client 初始化成功
- 但实际调用 DashScope 返回 404

**修复方向**：
1. 检查百炼工作空间 API Key 是否已过期/被撤销
2. 检查工作空间端点 URL 是否变更
3. 确认 qwen-plus/qwen-turbo 模型在工作空间中是否开通

### 🟡 P2：RAG 课标表缺失

`curriculum_standards` 表不存在于 PostgreSQL，导致课标对齐功能降级为 mock 数据。

**修复**：运行数据库迁移脚本初始化 RAG 表。

### 🟡 P2：URL 末尾斜杠导致 307 重定向

AI API 调用时某些请求带尾部 `/` 导致一次额外的 307 重定向，增加延迟。
