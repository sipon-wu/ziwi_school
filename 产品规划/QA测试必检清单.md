# QA 测试必检清单

> **用途**：QA 每次测试前通读一遍，强制对照检查。这些是从真实上线Bug中提炼的模式，不是理论清单。
> **使用方式**：逐项走查 → 标记 ✅/❌ → ❌ 项直接提 Bug。
> **来源**：老P 历次评审发现的问题汇总。

---

## 一、状态一致性（最容易漏，最致命）

### 1.1 组件状态与全局状态是否同步？

| 场景 | 检查点 |
|------|--------|
| 学科/年级切换 | 所有页面的下拉框是否跟随？出题页的推荐缺省值是否刷新？ |
| 知识图谱开关 | 开启/关闭时，所有引用图谱的页面是否一致响应？ |
| Token 配额用尽 | 各页面的 AI 生成按钮是否正确禁用？提示文案是否一致？ |

**历史案例**：
- ExerciseGenerator/LessonPlanEditor/SettingsPage 的学科/年级下拉框与全局 TeachingContext 不同步（已修复，2026-06-22）
- 知识图谱关闭后开关仍在 TopBar 显示（已修复，Phase 1）

### 1.2 多个入口修改同一数据时，是否双向同步？

| 场景 | 检查点 |
|------|--------|
| 头部切学科 vs 出题页切学科 | 两个方向都要测 |
| 设置页改教材 vs 教案页改教材 | 刷新后以谁为准？ |
| localStorage 持久化 | 关闭标签页重开，配置是否恢复？ |

## 二、交互非连续性（你以为步骤结束了，UI 还占着位置）

### 2.1 一次性决策是否在完成后自动折叠/隐藏？

| 场景 | 检查点 |
|------|--------|
| 命题用途选择 | 选完后 8 个大卡片是否自动收起为摘要行？ |
| 教案模板选择 | 选完后是否仍需占据等同面积？ |
| 新生引导 | 引导完成后，步骤条/遮罩是否消失？ |

**历史案例**：
- 命题用途卡片选定后不消失，占据 280px 首屏空间（已修复，2026-06-24）

### 2.2 面板/弹窗有无"死胡同"状态？

| 场景 | 检查点 |
|------|--------|
| 弹窗打开后 | 是否有关闭按钮？点击遮罩是否能关闭？ESC 是否生效？ |
| 空状态 | 没有数据时，该区域显示占位符还是空白？ |
| 全屏模式 | 进入全屏后，原来的操作条（已选列表、搜索、筛选）是否还在？ |

**历史案例**：
- 知识图谱全屏弹窗缺少已选知识点列表、搜索栏、单元选择器（已修复，2026-06-24）

---

## 三、副作用泄漏（A 操作意外影响 B）

### 3.1 用户缩放/滚动/拖拽状态是否被意外重置？

| 场景 | 检查点 |
|------|--------|
| 图谱节点点击 | 选中后画布是否被重置到默认缩放？ |
| 切换着色维度 | 用户之前的缩放位置是否保留？ |
| 切换布局模式 | 树状→螺旋→网状 来回切，缩放是否一直归零？ |
| 拖拽面板边缘 | 松手后是否触发了 fitView（把用户缩放覆盖了）？ |
| 难度滑块拖动 | 过滤节点后是否重置了视角？ |

**历史案例**：
- 树状/螺旋图谱点击任何节点 → 整个图谱销毁重建 → 缩放归零（已修复，2026-06-24）
- 面板拖拽调宽后调用 fitView() 覆盖用户缩放（已修复，2026-06-24）

### 3.2 列表操作是否影响排序/筛选状态？

| 场景 | 检查点 |
|------|--------|
| 删除一项后 | 列表是否回到默认排序？筛选条件是否丢失？ |
| 翻页后 | 回到第一页时顺序是否一致？ |
| 搜索后清空 | 是否回到搜索前的状态？ |

---

## 四、边界/空状态/溢出（正常流程测不到的死角）

### 4.1 尺寸边界

| 场景 | 检查点 |
|------|--------|
| 1366×768（笔记本） | 三栏是否挤压到不可用？面板间是否有最小宽度保护？ |
| 1920×1080（标准） | 面板宽度是否合理利用空间？是否出现大量空白？ |
| 2560×1440（宽屏） | 知识图谱面板是否过窄？是否仍固定 320px？ |
| 375×667（手机） | 底部导航 + 浮动按钮是否重叠？ |
| 面板拖拽到极限 | 最小 280px 时内容是否截断？最大 560px 时是否溢出？ |

**历史案例**：
- 1440px 下三栏溢出：1024+320+280=1624 > 1440（已修复：SidePanel 在 KnowledgePanel 打开时自动隐藏）
- 知识图谱面板 320px 在宽屏下过窄（已支持拖拽 280-560px）

### 4.2 内容边界

| 场景 | 检查点 |
|------|--------|
| 0 条数据 | 列表/图谱/卡片是否正确显示空状态？ |
| 1 条数据 | "清空"按钮是否仍然可用？删除后是否回到空状态？ |
| 极长文本 | 节点名称 20+ 字是否截断还是溢出？ |
| 已选 50 个知识点 | chip 列表是否滚动？清空按钮是否可见？ |

### 4.3 高度的"硬编码 420px"问题（通病）

| 场景 | 检查点 |
|------|--------|
| 任何画布/图表 | 高度是否用具体数值写死？容器变高后下半部分是否空白？ |
| 弹窗内嵌内容 | 是否用了 `height={420}` 而非 `h-full` / `flex-1`？ |
| 移动端 | 横屏/竖屏切换后高度是否自适应？ |

**历史案例**：
- 知识图谱全屏弹窗高度写死 420px，下半截空白（已修复，2026-06-24）

## 五、组件双写/重复实例（改一处漏另一处）

| 场景 | 检查点 |
|------|--------|
| 同一组件在多个页面出现 | 是否共用同一实例？还是各自独立渲染？ |
| 弹窗 + 内联 双模式 | 两种模式的功能是否对等？（弹窗缺搜索/内联缺全屏？） |
| 桌面 + 移动 双布局 | 移动端是否比桌面端少了功能？ |

**历史案例**：
- LessonPlanEditor 和 ExerciseGenerator 各有一份独立 KnowledgeGraph 实例（已修复，Phase 2 统一为 KnowledgePanel）
- 知识图谱全屏弹窗 vs 侧面板功能不对等（弹窗缺少已选列表/搜索/单元选择）

## 六、视觉一致性

### 6.1 颜色

| 场景 | 检查点 |
|------|--------|
| 草稿状态 | 所有页面的草稿标签颜色是否统一（yellow，不是 orange）？ |
| 已定稿/已完成 | 是否统一用 green？ |
| AI 预览态 | 是否统一 amber 边框 + "AI 预览" 标签？ |
| 学科色标 | 语文/数学/英语的标签颜色在所有列表页是否一致？ |

**历史案例**：
- DashboardHome 草稿颜色 orange→yellow 修复（Phase 1）

### 6.2 字号与间距

| 场景 | 检查点 |
|------|--------|
| 同一信息层级 | 标题/正文/辅助文字的字号是否全局一致？ |
| 同一类型按钮 | padding、圆角、hover 效果是否一致？ |
| 三种图谱布局 | 树状/螺旋/网状的节点大小和字号视觉上是否等大？ |

## 七、Z-轴与层级（弹窗套弹窗、遮罩叠遮罩）

| 场景 | 检查点 |
|------|--------|

## 检查清单（打勾版）

> 每轮测试前逐项过一遍，❌ 的直接提 Bug。

### 状态同步
- [ ] 头部切学科/年级 → 所有页面下拉框跟随
- [ ] 知识图谱开关 → 面板显隐 + TopBar 不残留开关
- [ ] localStorage 持久化 → 刷新后恢复

### 交互连续性
- [ ] 一次性决策完成后自动收起（命题用途、模板选择等）
- [ ] 弹窗有点击遮罩关闭 + X 按钮 + ESC
- [ ] 全屏模式保留所有操作条（搜索、筛选、已选列表）

### 副作用
- [ ] 点击图谱节点不重置缩放
- [ ] 切换布局/维度不重置缩放
- [ ] 拖拽面板宽度不触发 fitView
- [ ] 拖拽难度滑块不重置视图

### 边界
- [ ] 1366×768 下三栏不溢出
- [ ] 0 条数据有空状态占位
- [ ] 节点名 20+ 字不溢出
- [ ] 所有画布高度自适应（不是硬编码 420px）

### 组件一致
- [ ] 无重复实例（同一组件多处独立渲染）
- [ ] 弹窗/内联/移动 三种模式功能对等

### 视觉
- [ ] 草稿标签全部 yellow（不是 orange）
- [ ] AI 预览统统一 amber 边框

### Z-轴
- [ ] 多弹窗时 z-index 无冲突
- [ ] 移动端 fixed 元素无重叠

## 八、编译通过 ≠ 页面正常（血泪教训）

- [ ] **TypeScript/编译零错误不代表页面能渲染**。replace_in_file 可能静默失败（缩进不匹配），`firebase deploy` 可能成功但页面白屏。**必须打开浏览器看页面**。
- [ ] import 新增组件/图标后，确认页面加载无白屏（`ReferenceError: XXX is not defined` 是 tsc 不会捕获的运行时错误）
- [ ] 任何新增/修改 import 后，强制刷新预览页（Cmd+Shift+R）验证
