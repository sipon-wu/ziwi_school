## 产品概述

知微AI教学助手MVP开发正式启动。基于已确认的产品功能规格书（25页面+36弹层）、技术方案文档v1.4（3服务架构）、前端设计/编码规范，组建开发与测试团队，搭建三大代码库骨架，编写Sprint 1-2具体开发任务、测试用例矩阵、演示帐套仿真数据。

## 核心任务

1. **团队组建**：定义开发团队（前端/后端/AI）和测试团队角色与职责
2. **代码库骨架搭建**：Next.js前端 + Go Gin后端 + Python FastAPI AI服务三大代码库初始化，Docker Compose编排
3. **Sprint 1-2 开发**：基础设施搭建 + 用户认证系统 + 数据库迁移 + 教师工作台首页开发
4. **测试用例编写**：按25页面×5角色维度构建测试用例矩阵，覆盖正常流程/异常边界/权限校验
5. **演示帐套仿真数据**：准备虚拟学校"示例学校"的完整演示数据（3教师×3班级×50教案×200题目×60学生）

## 技术栈与架构

### 已确认技术选型

| 层 | 选型 | 版本 |
| --- | --- | --- |
| 前端 | Next.js + TypeScript | 14 (App Router) |
| 前端UI | Tailwind CSS + shadcn/ui | 3.4 / latest |
| 后端 | Go + Gin | 1.22 / 1.9+ |
| AI服务 | Python + FastAPI + LangChain | 3.12 / 0.3+ |
| 数据库 | PostgreSQL + pgvector | 16 / 0.7+ |
| 缓存 | Redis | 7 |
| 部署 | Docker Compose | 单机4C16G |

## 测试体系

### 测试分层

| 层级 | 工具 | 覆盖目标 |
| --- | --- | --- |
| 单元测试 | Go: go test / Python: pytest / TS: vitest | >=70% |
| 集成测试 | pytest + Go httptest | 核心API流程 |
| E2E测试 | Playwright | 5条关键用户路径 |
| 手动验收 | 测试团队按用例执行 | 25页面全部功能 |


### 测试用例矩阵维度

- 按页面（25个） × 按角色（5个） × 按场景（正常/边界/异常/权限）
- 弹层测试（36个二次确认弹窗）
- 小程序专项测试（拍照上传/Canvas签名/微信订阅消息）

## Agent Extensions

### SubAgent

- **code-explorer**
- 用途：探索当前code/目录下的已有文件，确认Logo素材位置及静态预览HTML结构，为前端脚手架搭建提供基线
- 预期结果：输出code/目录完整文件树及关键文件摘要

### Skill

- **playwright-cli**
- 用途：编写E2E测试用例时自动化浏览器验证登录流程、教案生成SSE流式渲染、批阅双栏交互等关键用户路径
- 预期结果：5条核心E2E测试脚本通过验证

### Skill

- **xlsx**
- 用途：生成测试用例矩阵Excel文件，按页面×角色×场景三维结构化组织，交付测试团队使用
- 预期结果：一份完整的测试用例矩阵.xlsx文件
